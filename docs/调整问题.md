# 🧠 一、整体架构（先看这个）

这四个模块不是独立的，其实是一条链路：

```
用户请求下载
   ↓
创建下载任务（入队列）
   ↓
Worker执行下载
   ↓
写入 videos / download_history
   ↓
触发 AI 分析（可选）
   ↓
写入 ai_tasks
   ↓
消耗 AI quota（VIP）
   ↓
支付系统补充额度 / VIP
```

👉 核心思想：

> **所有行为 → 入库 → 可计费 → 可追踪**

```

### 2️⃣ 创建队列

```
# core/queue.py
from redis import Redis
from rq import Queue

redis_conn = Redis(host="localhost", port=6379, db=0)
queue = Queue("video_tasks", connection=redis_conn)
```

------

### 3️⃣ 提交任务

```
# api/video.py

from app.core.queue import queue
from app.services.video_worker import download_video_job

@router.post("/download")
async def download_video(url: str):
    job = queue.enqueue(download_video_job, url)
    return {"task_id": job.id}
```

### 4️⃣ Worker执行

# 💰 四、VIP权限真正生效（关键）

你现在只是“有字段”，但没控制。

------

## 4.1 权限模型设计

```
class User:
    is_vip: bool
    vip_expire_at: datetime
    ai_quota: int
```

------

## 4.2 核心依赖（必须统一）

```
# core/permissions.py

def check_ai_quota(user):
    if user.is_vip:
        return True

    if user.ai_quota <= 0:
        raise Exception("AI quota not enough")

    user.ai_quota -= 1
```

------

## 4.3 在接口中使用

```
@router.post("/ai/summary")
def summary(user=Depends(get_current_user)):
    check_ai_quota(user)
```

------

## 4.4 下载权限

def require_vip(user):

        raise Exception("VIP required")

```

    cancel_url="...",

```

```
